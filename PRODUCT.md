# Product

## Register

product

## Users

Hackathon judges and blockchain developers reviewing the Casper Agentic Buildathon 2026 submission. Context: viewing a live demo dashboard to verify on-chain AI agent permission enforcement is real. Primary task: understand what the system does, trigger an action, read the on-chain result.

## Product Purpose

Casper Agent Leash is an operator dashboard for managing AI agent permissions on the Casper blockchain. An owner registers an AI agent with a spending cap and allowed action type; the smart contract enforces those limits on-chain. The dashboard drives the full lifecycle: register → fund → attempt transfer (allowed or blocked) → revoke. Every action produces a real, explorer-verifiable testnet deploy hash.

## Brand Personality

Precise · Powerful · Technical. The dashboard should feel like a control room, not a marketing page. Confidence through density and clarity. Reference: Dune Analytics — data-first, everything earns its place.

## Anti-references

- Glassmorphism-heavy DeFi dashboards (blur cards, glow gradients everywhere)
- SaaS cream/sand backgrounds with warm decorative tones
- Animated hero sections or particle backgrounds
- Neon/cyberpunk aesthetics — leans into crypto cliché rather than engineering credibility

## Design Principles

1. Data over decoration — every visual element must carry information or aid comprehension
2. State is sacred — every interactive element clearly communicates its current state
3. Trust through density — a full, precise layout signals engineering competence to judges
4. Color is semantic — green = allowed, red = blocked, violet = primary action only
5. The contract is the product — the dashboard is a window into on-chain enforcement, never the hero

## Accessibility & Inclusion

WCAG AA minimum. All interactive elements keyboard accessible. Sufficient contrast for all text (≥4.5:1 body, ≥3:1 large text). Reduced motion respected via prefers-reduced-motion.
